Bootstrap Templates

Orbiting since: Nov 10, 2012 by Stratos  | Tags: Bootstrap, css, best practices  | Comments (0)

A majority of Bootstrap theme developers seem to not understand the easy way to extend Bootstrap without altering its default behaviors.

A majority of Bootstrap theme developers seem to not understand the easy way to extend Bootstrap without altering its default behaviors.

The easiest way to customize Bootstrap happens to be painless, stress-free, and creative. How: by downloading the project from Github and writing your own customizations in the docs.css file. After all this is exactly what @mod and @fat have been doing for some time now, likely since the beginning of the project.

By editing only the last css file you can keep Bootstrap's default behavior in place. If you need to tweak anything, copy and paste the selector into your docs.css file. Doing so ensures that any customizations purchasers of your Themes know exactly where to tweak.

As it is I'm out of $60.00 spent for 3 templates. None of those 3 kept the default bootstrap in place. One came with 18,000 lines of code: 9,000+ selectors with 9,000+ comments. Buzzkill3 overkill. Somebody either teach me how to do a regex search and replace in Sublime Text 2, or theme developers can explore how bootstrap actually works.

Just because you have a really cool jQuery slider doesn't mean you have to edit out all traces of Carousel! And why in the world did you do away with the Responsive css code? And when I put them back in, why doesn't it work? Because you deleted it away!

For the love of all good things and Wisconsin, please, please, oh! pretty please create your own custom css file! I promise you that you'll achieve the same design and behavioral goals without sacrificing any of Bootstrap's defaults! Really.

And it's really easy!

After messing around with 3 different Themes the past 2 weeks, it took me a few hours to put this site's template together. It looks identical to a theme that I won't name because I want to encourage Theme developers to make it easy for their customers to tweak their designs.

That said, here's a few suggestions to keep in mind.

  1. Adjust the logo area
  2. Adjust the footer look and feel
  3. Don't fill up a "column" with 72 social media icons. See how it looks with just 2, 4 and 6
  4. Understand that some aspects of Admin themes can be the basis for neat derivatives.
    For example, this page is based on the home page of Bootstrap's jumbotron, just because.
  5. In that vein, I purchase Admin templates just because I love the Gallery options. It saves me time and work but if it doesn't seamlessly fit with my other Bootstrap implementations, then I can't use it.
  6. Muy importante to keep "back to the top" links since mobile browser will overtake desktop browing within a year.
  7. Stick with the structure of container/row/span

The last point contains the biggest impact on integrating your theme elements into my Bootstrap templates. I noticed that several themes mix that structure considerably, and ignore some basic HTML5 structure that can make huge SEO differences.

I'm talking about 2 things here.

1 container row span is the standard. container row span container row is needless and counterproductive. Just add an additional class to customize it. Often I have to because the theme's container has been edited to present some design element or another. container row span is layout classes only.

<div class="container">
  <div class="row">
    <div class="span3">
      <!--Sidebar content-->
    </div>
    <div class="span9">
      <!--Body content-->
    </div>
  </div>
</div>

2 Just one <h1> <h2> <h3> <etc.> tag per <section> Use <section> when you need to populate a page with several h1 tags. section id primary and section id secondary are the way to go.
